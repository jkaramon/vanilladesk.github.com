In many cases, tickets should be regularly updated, so the affected party is informed of any progress made. This task, however, becomes more difficult with higher number of tickets assigned to an Operator.

We have introduced the Ticket update reminder. This feature will allow you to specify intervals, in which tickets need to be updated and the system dispatches emails, reminding Operators to update them.

This new feature is ready to be used out-of-the-box, with only minimal setup needed.

The only action required is:

  • either specify the next update time manually, when adding a public update to a ticket,
  • or setup the update frequency in the Service-level agreement (SLA).

The pre-configured parts are:

  • a default template of the email, that will be sent
  • a list of recipients who will ne notified once the reminder is triggered
  • the percent of time remaining until the update is due

All above-mentioned settings can be adjusted any time.
